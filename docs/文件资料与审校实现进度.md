# 文件资料与审校实现进度

更新时间：2026-07-19

本文按《阶段上下文构建器》方案核对当前真实代码状态。状态只表示可运行实现，不以“Prompt 中出现字段”作为完成依据。

## 总体结论

当前已完成统一阶段上下文的 P0 主链：文件全文索引、阶段相关检索、生成使用快照、章节审校继承生成依据、审校补充检索和来源定位已经连通。

尚未完成的重点是：主大纲自动优先级与冲突决策、文献主题覆盖地图、人物/机构/时间线全书索引、资料变更影响追踪，以及可靠的观点—证据语义绑定。

## 状态说明

- **完成**：已有运行时调用、持久化或用户可见结果，并有回归测试。
- **部分完成**：主链存在，但精度、数据结构或阶段策略尚未达到目标方案。
- **未实现**：当前没有可工作的运行链。

## 审校证据链

章节审校当前按以下顺序构建证据：

1. 读取该章最近一次 `chapter` 生成快照中的实际使用资料。
2. 按待审正文进行 `review` 补充检索。
3. 同一来源重复命中时保留生成时记录，不用补充检索覆盖。
4. 审校 Prompt 明确区分 `chapter_generation` 与 `review_retrieval`。
5. 每条事实、引用类 finding 使用原文、问题说明和来源内容做匹配。
6. 命中时保存来源 ID、分块/文献 ID、页码或段落、使用阶段和生成 ID。
7. 未命中时写入 `evidence_gap=true`，不得假装已有资料支持。

## 当前验收覆盖

- 长文件文首、中部、文末唯一标记均能进入全文分块。
- 混合文件角色扫描覆盖文首、中部和文末，并携带页码/段落。
- 章节生成快照保留来源、分块、定位和生成记录。
- 章节审校合并来源时，生成时实际来源优先于审校重复命中。
- 审校 finding 能保存匹配来源定位；没有匹配时生成资料缺口标记。
- 待确认、失败和禁用文件不会进入阶段上下文。

## 下一实施顺序

1. 自动选择主大纲和处理版本冲突，移除正常流程对手动 `prepare_outline_context` 的依赖。
2. 建立文献主题覆盖地图与缺口，不把全部摘要注入大纲。
3. 增加人物、机构、任职和时间线事实索引，优先服务人物传记与全书审校。
4. 增加资料权限策略和资料变化影响追踪。
5. 将资料依据折叠区接入大纲和章节结果页。
