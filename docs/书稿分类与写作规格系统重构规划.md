# 书稿分类与写作规格系统重构规划

## 1. 文档目标

本规划解决现有 `book_type / style_type` 同时承担展示分类、数据库枚举、Prompt 路由和证据标准的问题。目标不是继续增加固定书型，而是把市场分类、具体写法和证据要求拆成三套可独立演进的资产。

本次上线不迁移书型数据库、不改变现有生成路由。下周版本仅完成统一资料搜索、运行时 Prompt 审计和非 AI 主题回归测试。本规划作为后续独立项目的设计基线。

## 2. 核心判断

出版分类调研方向正确，但平台货架分类不能直接等于写作策略：

- 微信读书、京东、当当的分类服务于发现和销售，同一层级可能混合学科、体裁、读者和用途。
- 出版社分类更接近选题和产品线，但仍带有机构自身的经营边界。
- 中图法是学科编目体系，适合领域归属和馆藏检索，不描述叙事视角、章节单元或读者关系。
- 热销书只能用于提炼候选写作特征，不能直接生成固定模板，否则会放大营销偏差、幸存者偏差和短期风格。

参考入口：[微信读书分类](https://weread.qq.com/web/category/100000)、[中图法基本大类](https://lib.phil.pku.edu.cn/node/34005)。实际调研须保存采集日期和页面快照摘要，不复制书籍正文。

## 3. 三套独立资产

### 3.1 MarketCategory

市场出版分类是开放、多选、可版本化的标签资产，只回答“这本书在市场和知识体系中归到哪里”。

建议字段：

| 字段 | 含义 |
| --- | --- |
| `id/version/status` | 稳定标识、版本和发布状态 |
| `platform` | 微信读书、京东、当当、出版社、中图法等 |
| `external_code` | 平台原始类目编码 |
| `name/parent_id/path` | 原始名称、父类和完整路径 |
| `aliases` | 同义类目和历史名称 |
| `dimension` | 学科、体裁、读者、用途、主题、销售货架 |
| `collected_at/source_url` | 采集时间和来源页 |
| `mapping_basis` | 映射理由、人工确认人和证据 |

一本书可关联多个市场分类，但必须分别标明主类、辅类和映射置信度。市场分类不直接选择 Prompt。

### 3.2 WritingSpec

`WritingSpec` 是决定具体写法的版本化契约，由用户意图、学科领域、目标读者、材料条件和编辑建议共同形成。建议至少包含：

- 产品承诺：读者读完后获得什么。
- 内容边界：覆盖范围、深度、明确排除项。
- 结构策略：篇章组织原则、章节单元、递进关系、是否允许非线性结构。
- 叙事策略：叙事视角、作者位置、时间结构、人物与案例使用方式。
- 论证策略：主张类型、反例要求、不确定性表达、事实与观点分层。
- 读者关系：默认知识水平、解释密度、术语引入方式、练习或行动装置。
- 语言约束：句段密度、抽象程度、语气、允许和禁止的修辞。
- 阅读装置：图表、边栏、案例、摘要、练习、时间线等的适用条件。
- 来源约束：关联的 `EvidencePolicy` 版本。
- 适用边界：哪些章节可例外、例外由谁确认。

`WritingSpec` 不保存一整套固定 Prompt。稳定 Prompt 只描述每个阶段的输入输出契约，运行时由编译器组合通用能力规则、当前规格、用户材料和阶段上下文。

### 3.3 EvidencePolicy

`EvidencePolicy` 按“主张类型”而不是按书型一刀切规定证据要求：

| 主张类型 | 最低来源要求 | 典型限制 |
| --- | --- | --- |
| 基础事实 | 可追溯公开来源 | 人名、日期、机构须交叉核对 |
| 学术理论 | 原始论文、权威专著或综述 | 不允许仅用搜索摘要支持复杂结论 |
| 政策法规 | 发布机关原文和现行版本 | 必须记录生效、修订或废止状态 |
| 统计数据 | 统计机关、数据集或报告原表 | 记录口径、年份、地区和单位 |
| 新闻事件 | 原始报道、当事方材料或多源报道 | 区分事件发生时间与报道时间 |
| 人物观点 | 可定位访谈、演讲、著作或公开账号 | 转述与直接引语分开 |
| 行业判断 | 报告、企业披露和多源数据 | 标注样本、商业立场和预测性质 |
| 作者经验 | 作者明确提供的材料 | 不伪装成普遍事实 |

策略应包含来源优先级、最少来源数、是否要求原始来源、冲突处理、引用格式、允许的通用知识边界和审校严重级别。

## 4. 热销书规格研究

### 4.1 样本设计

采用分层抽样，不按单个平台榜单直接归纳：

- 平台：微信读书、京东、当当及出版社公开榜单。
- 出版分类：覆盖主要市场类目和新增需求书型。
- 年份：近期样本与跨周期稳定样本并存。
- 作者类型：学者、记者、职业作者、当事人、机构作者等。
- 销量层级：头部、中腰部和普通出版物。
- 对照组：每个层至少加入非畅销但编辑质量稳定的图书。
- 规范对照：国家标准、出版社体例和人工编辑判断。

不得采集或保存受版权保护的书籍全文。研究输入以公开目录、元数据、合法试读片段、营销页信息和人工结构标注为主。

### 4.2 人工标注手册

每个样本记录可观察特征，不直接记录“照着写”：

- 全书结构和章节间关系。
- 叙事视角、时间顺序和作者位置。
- 章节内部的稳定单元与变化单元。
- 证据类型、引用密度和事实/观点边界。
- 读者称呼、解释前提和互动方式。
- 句段长度、术语密度、抽象层级和修辞频率。
- 图表、案例、边栏、练习、清单等阅读装置。
- 适用读者、适用主题和失效条件。

任何候选特征必须带样本量、跨组一致性、标注者一致率、置信度和适用边界。低样本结论只能进入实验规格，不进入默认生产规格。

## 5. 目标数据模型

建议新增以下核心实体：

- `market_categories` / `market_category_versions`
- `book_market_categories`
- `writing_specs` / `writing_spec_versions`
- `evidence_policies` / `evidence_policy_versions`
- `book_spec_bindings`
- `prompt_stage_contracts`
- `generation_snapshots`
- `spec_research_samples` / `spec_research_annotations`

`book_spec_bindings` 保存当前确认版本、确认人、确认时间和确认范围。规格变化产生新版本，不原地覆盖。

每次大纲、叙事宪法、章节、搜索计划和审校运行均保存：

```json
{
  "writing_spec_version_id": "uuid",
  "evidence_policy_version_id": "uuid",
  "stage_contract_version": "outline.v2",
  "compiler_version": "prompt-compiler.v1",
  "input_hash": "sha256",
  "compiled_prompt_hash": "sha256"
}
```

当绑定规格或上游产物的哈希变化时，下游产物标记为 `stale`，但不自动覆盖用户已编辑内容。

## 6. Prompt 编译器

编译顺序建议固定为：

1. 阶段角色和输出契约。
2. 安全、版权、事实和引用底线。
3. 当前 `WritingSpec` 的适用规则。
4. 当前 `EvidencePolicy` 的主张约束。
5. 全书与章节上下文。
6. 用户确认材料和负面约束。
7. 本次任务指令。

冲突优先级为：用户明确且合法的约束 > 已确认规格 > 证据底线 > 阶段默认能力。编译器必须输出冲突记录和规则来源，不能静默选择。

稳定 Prompt 不写“人物传记模板”“情感书模板”等整套无限分支。能力规则可组合，例如“时间线叙事”“概念递进”“案例驱动”“多声部材料”“训练与练习”等，并由规格声明适用范围。

## 7. 旧系统迁移

### 阶段 A：观测，不改路由

- 审计现有七类 Prompt 的领域假设。
- 为每次生成记录旧 `book_type/style_type`、实际 Prompt 和输入摘要哈希。
- 新建影子 `WritingSpec`，只生成和展示，不参与生产。

### 阶段 B：双写与人工确认

- 将旧七类映射为初始规格候选，保留映射理由。
- 用户可确认或调整规格；旧字段继续保存。
- 生成仍走旧路由，同时运行新编译器做差异比较。

### 阶段 C：按项目灰度切换

- 新项目使用规格路由，旧项目默认保持旧路由。
- 允许项目级回退，不允许把新规格反向压缩成错误旧枚举。
- 大纲、叙事宪法、章节、搜索和审校必须消费同一规格版本。

### 阶段 D：兼容字段退役

- `book_type/style_type` 变为只读兼容字段，不再决定 Prompt。
- 完成数据审计、回滚演练和旧项目导出后再考虑移除枚举约束。

## 8. 用户确认与变更影响

系统应先提出少量可理解的规格候选和差异，不要求用户理解内部字段。学科领域通常不超过三个，话题标签可多选；同一术语的解释优先受已确认学科领域约束。

用户修改规格时显示影响范围：仅未来章节、重生成大纲、重建叙事宪法、重新搜索证据或重新审校。高影响操作必须预览并确认，已有正文不得自动覆盖。

## 9. 验收标准

- 传记、情感、社科、技术、学术和实用类项目均可保存，不依赖新增数据库枚举。
- 同一市场分类可绑定不同 `WritingSpec`，同一规格可服务多个市场分类。
- 任一生成结果可追溯到规格、证据策略、阶段契约和编译器版本。
- 非 AI 主题的 Prompt 快照不得出现无上下文的人工智能、大模型或技术教材假设。
- 规格变化能可靠标记受影响产物，且不会静默覆盖用户内容。
- 研究结论均有样本量、置信度、适用边界和人工复核记录。

## 10. 本周边界

本周只实施：统一资料搜索、真实连接器执行记录、双入库、Prompt 审计、非 AI 回归测试和本规划文档。

本周不实施：书型数据库迁移、旧七类删除、Prompt 编译器切换、自动重生成或现有项目规格回填。后续实施必须单独建迁移计划、灰度开关和回滚方案。
