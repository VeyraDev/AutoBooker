分类规则：
1. 操作步骤、用户路径、从 A 到 B 的流程 → flowchart / process_flow
2. 是否/判断/分支/达标/不达标 → decision_flow / decision_tree
3. 系统模块、层级、服务、数据库、服务化架构 → architecture / system_architecture
4. 数据流、采集清洗存储 → dataflow / process_flow
5. 调用过程、客户端请求服务端 → sequence / process_flow
6. 两个或多个对象优劣/差异 → comparison / comparison_matrix
7. 四象限、二维矩阵 → comparison / comparison_matrix
8. 分类体系、思维导图、组织架构、上下级 → taxonomy / taxonomy_map
9. 知识图谱、关系网络、概念之间的语义关系 → taxonomy / concept_diagram
10. 时间演进、路线图 → timeline / timeline_roadmap
11. 章节总结、多个信息块 → taxonomy / infographic
12. 具象场景、人物、氛围 → illustration / scene_illustration
13. 有真实数值/统计趋势 → data / chart
14. 截图、界面截图、产品界面但未上传图片 → screenshot / screenshot

不要输出任何额外 subtype；必须归并到上面的 V3 图类。

重要：
- title 只给短图题，禁止复制版式说明
- "X连接前N个服务"等关系句不影响分类，不要写进 title
- reason 必须说明分类依据

语义净化规则（必须严格执行）：
- 描述中出现"X连接前N个/所有Y""X通过Z通知Y""X包含N个模块"等聚合关系句，这些句子描述的是 edges/connections，不是节点名。不要将整句或其任何片段写进 diagram_subtype 或 title。
- title 只给语义图题（如"系统架构""甲乙流程"），禁止包含"共X个""前N个""通过某组件"等关系描述词。
- 若描述中明显是"说明如何布局"（如"用方框和箭头展示""左边是A右边是B""用箭头连接以下步骤"），这是版式说明，不影响 diagram_family/subtype 判断，直接忽略。

【diagram_type 与 title 约束】
- title 是图的"名字"，不是图的"描述"。只写 4-8 字的名词短语（"甲乙流程""系统架构""调用链"）。
- 禁止在 title 中出现：动词（连接/通过/展示）、计数词（三个/前两个/共五个）、位置词（左侧/右边）。
- 若描述中同时出现"步骤/流程/阶段"和"模块/服务/架构"，优先判断用户真正想看的是哪一个：
    如果描述以"A→B→C"或"先做X再做Y"为主 → workflow/process_flow
    如果描述以"X包含Y Z，Y调用Z"为主 → architecture/system_architecture
- "甲模块通过消息中介异步通知乙模块"是一条 relation，不影响 diagram_type 判断，不进 title。
