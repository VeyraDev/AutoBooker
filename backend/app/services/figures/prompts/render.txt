你是渲染规划模块。根据语义结构，决定每个节点如何显示。不再理解原始描述。

语义输入：{semantic_json}

输出 JSON：
{
  "layout": "LR|TB|LAYERED|GRID|RADIAL",
  "canvas": {"width": 960, "height": 560},
  "nodes": [
    {
      "entity_id": "e1",
      "label": "API网关",
      "shape": "rounded|box|diamond|pill|cylinder",
      "color": "blue|teal|amber|purple|coral|gray|green",
      "size": "sm|md|lg",
      "group_id": "g1",
      "level": 0,
      "column": 0
    }
  ],
  "edges": [
    {
      "from": "e1",
      "to": "e2",
      "label": "HTTP",
      "style": "solid|dashed",
      "routing": "LR|TB|side|curved"
    }
  ],
  "groups": [
    {
      "id": "g1",
      "label": "入口层",
      "color": "blue",
      "node_ids": ["e1"]
    }
  ]
}

【layout 选择规则】
- diagram_type=flow + 纯线性链（每个节点最多 1 条入边和 1 条出边，无并行/汇合/反馈）且 entities≤5 → LR
- diagram_type=flow + 有并行分支、汇合点、decision 节点或反馈边 → TB
- diagram_type=flow + entities>5 → TB
- diagram_type=architecture + 有 groups → LAYERED（groups 作为层，每层内节点横排）
- diagram_type=comparison → GRID
- diagram_type=tree → RADIAL（根节点居中，子节点向外辐射）
- diagram_type=timeline → LR
- diagram_type=network → TB（按 importance 分层）
- diagram_type=infographic → GRID

【nodes 规则】
- label = entity.name，不做任何修改
- shape 按 entity.type 映射：
    service/api → rounded
    user/external → pill
    data/database → cylinder
    process → box
    decision → diamond
    group/module → rounded
- color 按 group 分配：同一 group 内节点颜色相同，不同 group 用不同颜色
    无 group 时按 entity.type：service→blue，data→teal，user→purple，decision→amber，其余→gray
- size：importance=1→sm，importance=2→md，importance≥3→lg
- level/column：严格按 relations 拓扑推断，不要凭直觉摆放
    flow/timeline 线性链：level 递增，column=0
    并行分支：同 level 不同 column（如 A、B 在 level=0 的 column=0/1）
    汇合节点：level = max(上游 level)+1，column=0
    反馈边：不改变目标节点 level，但 edge.routing=curved
    architecture LAYERED：group 序号即 level，group 内节点按 column 排列
    GRID：按行列均匀分配

【edges 规则】
- label = relation.label（可空）
- style：relation.async=true → dashed；否则 → solid
- routing 规则（关键）：
    layout=LR → 所有 edge 用 LR（出发点=节点右边中点，到达点=节点左边中点）
    layout=TB → 跨 level 的 edge 用 TB（底部→顶部）；同 level 不同 column 用 side（右边→左边）
    layout=LAYERED → 跨 group 的 edge 用 TB；同 group 内用 side
    feedback/return 类型的 edge 用 curved（从右侧绕回）

【canvas 计算】
- width = max(800, entity数量 × 180, group数量 × 260)，上限 1400
- height = max(500, level层数 × 160 + 120)，上限 1000

【禁止事项】
- 不得修改 entity.name 作为 label
- 不得新增 entities 中不存在的节点
- 不得删除任何 entity（即使你认为它不重要）
